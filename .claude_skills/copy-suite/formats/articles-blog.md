# Articles, Blog Posts & Newsletters

## Workflow

1. Identify the format (article, blog post, newsletter edition)
2. Name the one job and target reader
3. Choose the structure based on content type
4. Write the headline and lead first
5. Draft the body
6. Run the 4-pass edit (see Editing Process below)
7. Run the output quality gate (see SKILL.md Quality Gate section)

## Article Architecture

| Structure | Best For | How It Works |
|-----------|----------|-------------|
| **Inverted pyramid** | News, announcements, updates | Most important info first, details descend in priority |
| **Narrative arc** | Stories, case studies, opinion | Setup → tension → resolution → takeaway |
| **Modular** | How-to guides, listicles, tutorials | Independent sections that each deliver value |

### Length Guidelines
- **1500 words:** One clear argument with 2-3 supporting points. Tight.
- **3000 words:** Comprehensive treatment with examples, data, and nuance.
- **5000+ words:** Definitive guide. Needs section headers, table of contents, and internal navigation.

## Headlines

**Formulas that work:**
- Number + adjective + noun + promise: "7 Unconventional Strategies That Actually Grow Revenue"
- How to + outcome: "How to Write Copy That Doesn't Sound Like a Robot"
- Question the reader answers "yes" to: "Are You Making These Pricing Page Mistakes?"
- Bold claim: "Everything You Know About Email Subject Lines Is Wrong"

**Title length:** 40-60 characters for SEO. Under 70 to avoid truncation in search results.

**Test:** If the headline doesn't make someone decide to read OR skip within 2 seconds, it's too vague.

## The Lead (Opening)

The first 100 words determine if the other 2000 get read.

| Lead Type | Example Opening | Best For |
|-----------|----------------|----------|
| **Anecdotal** | "Last March, I watched a $2M launch fail..." | Stories, case studies |
| **Statistical** | "73% of landing pages have a CTA problem." | Data-driven pieces |
| **Question** | "What if everything you know about X is wrong?" | Contrarian pieces |
| **Bold statement** | "SEO is dead. Long live search." | Opinion, thought leadership |
| **Scene-setting** | "It's 3am. You're staring at a blank page..." | Personal, relatable pieces |

## SEO-Aware Writing

- **People first, search engines second.** Write naturally. Keyword-stuffed copy reads like spam and tanks engagement.
- **Primary keyword in:** H1 headline, first 100 words, one H2, meta description.
- **Header structure:** H1 (title) → H2 (main sections) → H3 (subsections). This is navigation for readers AND search engines.
- **Internal links:** 3-5 per article to related content. Natural anchor text, not "click here."
- **Meta description:** 150-160 characters. Include primary keyword. Write it like a hook, not a summary.

## Newsletter Craft

### Subject Lines
- 30-50 characters (mobile-optimized)
- Curiosity or benefit, never both
- Test: would you open this from a stranger?
- Never: ALL CAPS, excessive punctuation!!!, "Don't miss this"

### Structure
1. Personal greeting or hook (1-2 sentences)
2. Main content (the value they subscribed for)
3. One clear CTA (not three competing ones)
4. Sign-off that feels human

### Welcome Email
The most-opened email you'll ever send. Use it to:
- Deliver the promised value immediately
- Set expectations (what they'll get, how often)
- Tell a brief origin story (build connection)
- Ask one question (engagement + data)

## Readability Techniques

- **Bucket brigade:** Short transitional phrases that pull the reader forward. "Here's the thing." "But it gets better." "That's not all."
- **Open loops:** Mention something you'll address later. Reader stays to see it resolved.
- **Sentence variety:** Mix short (5 words), medium (15 words), and long (25+ words). Monotonous length puts readers to sleep.
- **Subheadings as parallel narrative:** Someone scanning only the subheadings should get the gist of the article.

## Editing Process (4 Passes)

1. **Structure pass:** Is the flow logical? Does each section earn its place? Cut entire sections if needed.
2. **Clarity pass:** Is every sentence clear on first read? Rewrite anything that requires re-reading.
3. **Cut pass:** Remove 20-30%. Kill adverbs, filler phrases, throat-clearing sentences, redundant points.
4. **Read aloud pass:** Read the entire piece out loud. Every stumble is a revision opportunity.
