# Thread Copy

## Workflow

1. Name the one job of this thread (educate, persuade, tell a story, establish authority?)
2. Choose the thread type
3. Write the hook tweet first (this determines everything)
4. Outline the arc before writing the body
5. Write the close and CTA
6. Fill in the body tweets
7. Read the whole thread as a reader would. Does each tweet make you want the next?
8. Run the output quality gate (see SKILL.md Quality Gate section)

## Thread Architecture

Every thread has four parts:

| Part | Tweets | Job |
|------|--------|-----|
| **Hook** | 1 | Stop the scroll. Promise value. Signal "this is worth reading." |
| **Bridge** | 2-3 | Context. Why should they care? Set up the payoff. |
| **Body** | 5-30+ | Deliver the value. Each tweet is a complete thought that leads to the next. |
| **Close** | 1-2 | CTA + recap of value. The follow/bookmark/share moment. |

## Hook Tweet Formulas

The hook tweet is different from a single tweet hook. It must promise value AND signal "this is a thread worth your time."

- "I spent [time] studying [topic]. Here's what most people miss:"
- "Most people think [common belief]. They're wrong. Here's the truth:"
- "[Number] lessons from [impressive/relatable thing]:"
- "[Surprising personal result] in [timeframe]. The playbook:"
- "The [topic] nobody is talking about:"

**Test your hook:** Remove the thread. Does the hook tweet stand alone as interesting? If yes, it'll work as a thread opener.

## Thread Types

| Type | Structure | Best For |
|------|-----------|----------|
| **Educational** | Concept → breakdown → examples → application | Building authority |
| **Story** | Setup → conflict → turning point → resolution → lesson | Emotional connection |
| **Listicle** | Intro → numbered items → close | Bookmarks, shareability |
| **Case study** | Problem → approach → result → lessons | Credibility, proof |
| **Contrarian** | Common belief → why it's wrong → better approach | Engagement, replies |
| **How I did X** | Context → steps → results → what I'd do differently | Practical value |
| **Curated** | Theme → item 1 → item 2... → why this matters | Community building |

## Narrative Thread Techniques

- **Each tweet is a cliffhanger.** End tweets with tension. "But then..." "What I didn't expect..." "Here's where it gets interesting."
- **Vary the pacing.** Short punchy tweets for impact, longer ones for context. Never three long tweets in a row.
- **One idea per tweet.** If a tweet has two ideas, split it or cut one.
- **The mid-thread energy dip is real.** Tweets 5-8 are the danger zone. Put a mini-hook or surprising stat there.
- **Number only when it helps.** Educational/list threads: number them. Story threads: don't. Numbering a story kills the narrative flow.

## Mega-Thread (25+ tweets)

- Use section headers (bold or emoji) as rest stops
- Include a "thread so far" recap at the midpoint
- Vary content type within sections (text, observation, question, quote)
- Progress signals: "We're getting to the good part" or section numbers (Part 1/3)
- The energy must RISE toward the end, not fade

## Engagement Positioning

| Position | Optimized For | Strategy |
|----------|--------------|----------|
| First tweet | Impressions | Maximum hook power |
| Middle tweets | Bookmarks | Highest-value insights here |
| Last tweet | Follows + shares | "If you found this useful, follow for more [specific value]" |

## Repurposing

A good thread is raw material for:
- Blog post (expand each tweet into a paragraph)
- Newsletter edition (add personal context)
- LinkedIn post (condense to key insights)
- Short video script (each tweet = one talking point)

Write threads with repurposing in mind. If each tweet is a standalone insight, repurposing is trivial.
