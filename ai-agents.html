The problem

Full keys or nothing isn't good enough.

Today's agents hold full private keys or none at all. There's no way to scope what an agent can do, enforce spending limits, or require human approval for high-stakes actions.

Key exposure risk

One compromised agent leaks keys to everything it controls.

No spending boundaries

Agents can drain entire balances with no programmatic limit.

No escalation path

When an action needs human judgment, there's no built-in mechanism.

Guardrails

Cryptographic permissions, not API rules

Ika enforces agent boundaries at the signing layer. Not in middleware, not in a dashboard — in the cryptography itself.

Spending limits

Per-transaction and daily volume ceilings. Enforced before signing, not after.

Contract allowlists

Restrict which protocols and addresses an agent can interact with.

Session windows

Time-scoped delegation that expires automatically. No revocation needed.

User escalation

High-stakes actions route to human co-signature. Configurable thresholds.
